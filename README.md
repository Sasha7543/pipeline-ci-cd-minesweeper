## Code

Реалізація розділена на модулі:

- `src/minesweeper/models.py` — чиста логіка гри без залежності від Pygame;
- `src/minesweeper/game.py` — графічний інтерфейс Pygame;
- `src/minesweeper/__main__.py` — запуск гри та обробка аргументів командного рядка;
- `tests/test_board.py` — базові тести правил гри.

У грі реалізовано:

- відкриття клітинок лівою кнопкою миші;
- встановлення прапорця правою кнопкою миші;
- перший клік не потрапляє на міну;
- автоматичне відкриття порожніх зон;
- стани перемоги та поразки;
- перезапуск через клавішу `R` або клік по верхній панелі;
- налаштування через `argparse`.

## Build

### Встановлення залежностей

```powershell
py -3.11 -m pip install -r requirements.txt
```

На цьому комп'ютері команда `python` відкриває системний ярлик Python 3.14, для якого `pygame` може не встановлюватися. Тому для запуску використовується Python 3.11 через команду `py -3.11`.

### Запуск гри

```powershell
py -3.11 -m src.minesweeper
```

Або з параметрами:

```powershell
py -3.11 -m src.minesweeper --difficulty easy --background "#eef6ff" --cell-size 34
```

Інші варіанти запуску для демонстрації `argparse`:

Легка складність зі світлим фоном:

```powershell
py -3.11 -m src.minesweeper --difficulty easy --background "#eef6ff"
```

Звичайна складність зі збільшеними клітинками:

```powershell
py -3.11 -m src.minesweeper --difficulty normal --cell-size 38
```

Складний режим:

```powershell
py -3.11 -m src.minesweeper --difficulty hard
```

Власний розмір поля та кількість мін:

```powershell
py -3.11 -m src.minesweeper --rows 10 --cols 14 --mines 20
```

Власне поле з іншим кольором фону:

```powershell
py -3.11 -m src.minesweeper --rows 8 --cols 12 --mines 15 --background "#f7f3e8"
```

Велике поле з меншими клітинками:

```powershell
py -3.11 -m src.minesweeper --rows 16 --cols 24 --mines 60 --cell-size 26
```

Цими прикладами я демонструю, що через `argparse` можна змінювати складність, розмір поля, кількість мін, розмір клітинки та колір фону.

### Запуск тестів

```powershell
$env:PYTHONPATH="src"
py -3.11 -m unittest discover -s tests
```

## Архітектурні принципи

- **DRY:** логіка роботи з клітинками і сусідами винесена в окремі методи.
- **KISS:** правила гри реалізовані простими класами `Board` і `Cell`.
- **SOLID:** модель гри не залежить від Pygame, тому її можна тестувати окремо від інтерфейсу.
